# ADR-0002: Десктопный Shell — Tauri

**Статус**: Принято  
**Дата**: 2026-05-31

---

## Контекст

СТРАННО — прежде всего десктопное приложение (macOS, Windows, Linux).
Нужен shell, который:
- Рендерит UI (viewport, node-граф, панели)
- Вызывает нативное Rust-ядро без overhead
- Кросс-платформен
- Имеет нативный доступ к файловой системе, GPU, аудио

---

## Варианты

### Electron
- ✅ Зрелый, огромное комьюнити
- ❌ Chromium в бандле = +150MB, высокое потребление RAM
- ❌ JS ↔ Rust коммуникация через IPC с сериализацией

### Tauri
- ✅ Rust-backend нативно
- ✅ Системный WebView (нет Chromium) = малый бандл (~10MB)
- ✅ Прямые вызовы Rust из JS через `invoke()`
- ✅ Нативный доступ к GPU через wgpu из Rust-слоя
- ⚠️ Разные WebView-движки на разных ОС (Blink/WebKit) — нужно тестирование

### Qt
- ✅ Нативный UI, отличная производительность
- ❌ Сложная лицензия (LGPL/коммерческая)
- ❌ Отдельный язык (QML) или C++ — усложняет стек

---

## Решение

**Tauri v2** как десктопный shell.

- **Rust-сторона**: nazrOS CORE, рендер, файловые операции, аудио
- **WebView-сторона**: React + TypeScript UI, design system СТРАННО
- **Коммуникация**: Tauri `commands` (Rust) ↔ `invoke()` (JS)
- **Viewport**: нативный Vulkan-рендер в отдельном окне / offscreen surface,
  результат → WebView через shared texture или IPC с минимальной задержкой

---

## Последствия

- Viewport рендерится нативно (не в WebView) — это нормально и правильно
- Вся бизнес-логика UI остаётся в React: panels, timelines, node graphs
- WebGPU в WebView — для лёгкого превью и виджетов
- Tauri updater — автообновления из коробки
