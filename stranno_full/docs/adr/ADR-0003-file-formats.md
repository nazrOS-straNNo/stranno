# ADR-0003: Форматы файлов nazrOS — кириллические расширения

**Статус**: В процессе (названия утверждаются по ходу реализации)  
**Дата**: 2026-05-31

---

## Контекст

СТРАННО создаёт русскоязычное комьюнити и экосистему.
Кириллические расширения — культурный идентификатор платформы,
не техническое решение (файловая система их поддерживает).

Форматы должны быть:
- Бинарными (производительность, размер)
- Версионированными (обратная совместимость)
- Документированными (открытая спецификация для комьюнити-инструментов)

---

## Принципы именования

1. Короткие (2–4 символа)
2. Интуитивно понятны носителю русского языка
3. Не конфликтуют с существующими расширениями

## Бинарный формат `.нзр` — структура (v0.1)

```
Заголовок (32 bytes):
  [0..4]   Magic: "НЗР\0" (UTF-8 байты + null)
  [4..8]   Версия формата: u32 little-endian
  [8..16]  UUID сцены: 128-bit
  [16..24] Timestamp создания: u64 Unix epoch ms
  [24..32] Флаги: u64 (сжатие, шифрование и т.д.)

Секции (TLV — Type-Length-Value):
  Type:   u16 (код секции)
  Length: u32 (размер в байтах)
  Value:  [u8] (данные секции, опционально zstd-сжатые)

Коды секций (черновик):
  0x0001  ГРАФ      — Scene Graph (дерево объектов)
  0x0002  ГЕОМ      — Геометрия (меши, кривые)
  0x0003  МАТЛ      — Материалы
  0x0004  АНМ       — Анимации
  0x0005  ФИЗ       — Физические данные
  0x0006  СВЕТ      — Световые настройки
  0x0007  КАМ       — Камеры
  0x0008  МЕТА      — Метаданные (автор, теги, превью)
  0x00FF  КОН       — Конец файла + CRC32 checksum
```

## Сетевые протоколы nazrOS (черновик)

Внутренние протоколы платформы также получат кириллические идентификаторы:

| Протокол | Назначение |
|----------|-----------|
| `НЗР-СИНХ` | Real-time синхронизация сцены между участниками |
| `НЗР-РНDR` | Протокол отправки задач на рендер-ферму |
| `НЗР-АКТВ` | Протокол передачи ассетов |
| `НЗР-ЖВЕ` | Heartbeat / health-check нод кластера |

*(Спецификации протоколов — отдельные ADR по мере реализации)*

## Последствия

- Парсер `.нзр` реализуется в `src/core/formats/` на Rust
- Все форматы имеют версию — обратная совместимость обязательна
- Спецификации публикуются открыто для комьюнити-инструментов
- Конвертеры из/в FBX, GLTF, Alembic, USD — отдельный модуль экспорта
